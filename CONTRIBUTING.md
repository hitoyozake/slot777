# 基本ルール
- masterブランチへの直接push禁止
- 必ずブランチを作って作業を行う
- 実装項目をIssueに立てて、ブランチを紐づける。
- PR(Pull Request)を通してmasterブランチにマージする
- feature, fix ブランチの2種類を基本とする
- 実装時にはドキュメントの整備を行う(仕様等)
- ドキュメントは`docs/`に格納する
- PR, Issueには適切なラベル付けを行う

# ブランチ命名規則
- fatureブランチは`feature/****`という名前にする
    - 例: feature/create-entry-point-for-api-XXX
- fixブランチは`fix/****`という名前にする
    - 例: fix/replace-typo-in-user-manual
