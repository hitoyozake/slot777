# ゲーム仕様書

バージョン: 1.0（Phase1・Phase2対応）

---

## 1. ゲーム概要

ブラウザで動くスロットマシン。KUMONの知育玩具「くるくるチャイム」のベル音をマイクで検出し、フィーバーゲージを操作してゲームを有利に進める。

---

## 2. リール仕様

### 2-1. リール構成

- 縦3行 × 横3列 の計9マスのグリッド
- 各列が独立したリールとして機能する（左・中・右）

```
┌────┬────┬────┐
│ 上 │ 上 │ 上 │  ← 行0
├────┼────┼────┤
│ 中 │ 中 │ 中 │  ← 行1（メインライン）
├────┼────┼────┤
│ 下 │ 下 │ 下 │  ← 行2
└────┴────┴────┘
  列0   列1   列2
```

### 2-2. シンボル一覧

| シンボル | 表示 | 通常配当（×BET）| 出現頻度（通常） |
|----------|------|----------------|----------------|
| 7 | 7️⃣ | 100 | 低（1/20） |
| BELL | 🔔 | 30 | やや低（1/20） |
| CHERRY | 🍒 | 10 | 中（4/20） |
| WATER | 💧 | 5 | 中（3/20） |
| BAR | 🎰 | 3 | やや高（4/20） |
| BLANK | （空白） | 0 | 高（8/20） |

> シンボルストリップは20コマ構成。停止位置はランダムに決定する。

### 2-3. フィーバー時のリール

フィーバー状態では専用の `FEVER_REEL_STRIP` に切り替わる。高配当シンボルの出現割合が増加し、BLANK が減少する。

| シンボル | フィーバー時の出現頻度 |
|----------|----------------------|
| 7 | 通常比 約2倍 |
| BELL | 通常比 約3倍 |
| CHERRY | 通常比 約2倍 |
| WATER | 通常比 約1.5倍 |
| BAR | 通常比 約1.5倍 |
| BLANK | 排除 |

## 3. 当たり判定（ペイライン）

### 3-1. 判定ライン（5ライン固定）

```
ライン① ━━━━━  上段横（行0: 列0→列1→列2）
ライン② ━━━━━  中段横（行1: 列0→列1→列2）
ライン③ ━━━━━  下段横（行2: 列0→列1→列2）
ライン④ ╲       左上→右下斜め（行0列0 → 行1列1 → 行2列2）
ライン⑤ ╱       左下→右上斜め（行2列0 → 行1列1 → 行0列2）
```

### 3-2. 当選条件

- **3マス完全一致**（同一ライン上の3シンボルがすべて同じ）
- BLANKは当選対象外
- 複数ラインが同時に当選した場合、すべての配当を合算する

### 3-3. 配当計算

```
獲得クレジット = シンボル配当値 × BET額
```

同一スピンで複数ライン当選した場合は合算。

**例:** BET=3 でBELLが2ライン揃った場合 → 30 × 3 × 2 = 180 クレジット

## 4. クレジット仕様

### 4-1. 初期値

- ゲーム開始時のクレジット: **100**

### 4-2. BET

| BETボタン | BET額 | 条件 |
|-----------|-------|------|
| BET 1 | 1 | 残高 ≥ 1 |
| BET 3 | 3 | 残高 ≥ 3 |
| MAX | 10 | 残高 ≥ 10（残高が10未満の場合は残高をBET額とする） |

- SPINごとにBET額を残高から差し引く
- 当選時は `シンボル配当値 × BET額` を残高に加算する

### 4-3. ベルボーナス

- ベル音検出（マイク or シミュレータ）のたびに **+1 クレジット** 加算
- フィーバー状態中も加算される

### 4-4. クレジット不足

- 残高 < BET額 の場合、SPINボタンは無効（disabled）になる
- ゲームオーバー画面は現時点では未実装（Phase3以降で対応予定）

## 5. フィーバーゲージ仕様

### 5-1. ゲージの基本動作

| 項目 | 値 |
|------|-----|
| ゲージ範囲 | 0〜100 |
| ベル検出時の上昇量 | +20 |
| 自然減衰速度 | -5 / 秒（100ms ごとに -0.5） |

### 5-2. フィーバー発動条件

- ゲージが **100** に達した瞬間にフィーバー状態へ移行する

### 5-3. フィーバー中の挙動

- リールが `FEVER_REEL_STRIP` に切り替わる（高確率モード）
- 筐体の枠が赤く光るアニメーションが発動する
- ゲージの減衰は停止する
- ベル検出時のゲージ上昇も停止する（フィーバー中はゲージ操作不可）
- フィーバー中にSPINすると、結果表示後 800ms でフィーバーが終了する

### 5-4. フィーバー終了

- フィーバー中にSPINを1回行うと終了する
- 終了時にゲージを 0 にリセットし、通常リールに戻る

## 6. マイク入力仕様

### 6-1. 検出アルゴリズム

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| 検出帯域 | 1,000Hz〜4,000Hz | くるくるチャイムのベル音が主に分布する帯域 |
| 振幅閾値 | 80 / 255 | AnalyserNode の getByteFrequencyData スケール |
| FFTサイズ | 2,048 | 周波数分解能の設定 |
| クールダウン | 500ms | 検出後の再検知抑制期間 |

### 6-2. マイクON/OFF挙動

| 状態 | シミュレータボタン | 検出方法 |
|------|------------------|----------|
| マイクOFF（初期状態） | 表示 | ボタン操作 |
| マイクON | 非表示 | マイク入力（自動） |

- マイクONボタンをタップするとブラウザのパーミッションダイアログが表示される
- パーミッションが拒否された場合、エラーメッセージを表示してシミュレータ継続
- マイクOFFボタンをタップすると AudioDetector を停止し、シミュレータに戻る

### 6-3. 不正について

- マイク経由での不正入力（録音MP3再生など）は **考慮しない**
- 誤検知についても軽度であれば許容する設計とする

## 7. ゲーム状態遷移

```
         ┌──────────────────────────────┐
         ▼                              │
       IDLE ──[SPIN押下]──▶ SPINNING    │
         ▲                    │         │
         │              [停止完了]       │
         │                    ▼         │
         │                RESULT        │
         │                    │         │
         │            [配当処理完了]     │
         │                    │         │
         └────────────────────┘         │
                                        │
       IDLE ──[ゲージ100%]──▶ FEVER ───┘
                               │
                         [SPIN完了後800ms]
```

| 状態 | 説明 | 可能な操作 |
|------|------|-----------|
| IDLE | 待機中 | SPIN、BET変更、ベル検出 |
| SPINNING | リール回転中（1,200ms） | なし（入力ロック） |
| RESULT | 当否判定・表示中 | なし（自動遷移） |
| FEVER | フィーバー演出中 | SPIN |

## 8. UI挙動仕様

### 8-1. 画面レイアウト（縦向き基準）

```
┌─────────────────────┐
│  🎤 マイクバー       │
├─────────────────────┤
│  FEVER GAUGE [████░] │
├─────────────────────┤
│  ┌───┬───┬───┐      │
│  │   │   │   │      │
│  ├───┼───┼───┤      │
│  │   │   │   │      │ ← リールグリッド
│  ├───┼───┼───┤      │
│  │   │   │   │      │
│  └───┴───┴───┘      │
│  CREDIT: 100  BET: 1 WIN: 0 │
│  [BET1] [BET3] [MAX] │
│  [   SPIN ▶  ] [🔔ベル] │
│  メッセージ表示エリア  │
└─────────────────────┘
```

### 8-2. 各UIパーツの挙動

#### マイクバー
- 常時表示
- マイクOFF時: 「🎤 マイクOFF（シミュレータ使用中）」グレー表示
- マイクON試行中: 「🎤 接続中...」ボタンはdisabled
- マイクON成功: 「🎤 マイクON（ベル音を検出中）」、ボタンは「マイクをOFFにする」に変化
- マイクON失敗: 「🎤 マイク取得失敗（シミュレータを使用）」

#### フィーバーゲージ
- ゲージ値に応じてバーの幅が変化（0〜100%）
- 右端に現在の % 値を数値表示
- 100%到達時: バーが点滅アニメーション
- フィーバー中: 点滅を維持

#### リールグリッド
- SPINNING中: 全セルにブラー（blur）アニメーションを付与
- 停止後: アニメーション解除、シンボルを表示
- 当選ライン上のセル: 金色ボーダー＋緑背景のフラッシュアニメーション（3回）

#### SPINボタン
- 通常時: 有効（金色グラデーション）
- SPINNING / RESULT 中: disabled
- 残高 < BET の場合: disabled
- FEVER中: 有効（フィーバースピン可能）

#### ベルシミュレータボタン
- マイクOFF時のみ表示
- マイクON時: `display: none`

#### BETボタン
- 選択中のBETボタンに金色ボーダーを表示
- SPINNING中も操作可能（次のスピンに反映される）

#### メッセージエリア
- WIN時: 「WIN! +{配当}」または「🎉 JACKPOT!! 🎉」（配当100以上）
- はずれ時: 「はずれ…」
- ベル検出時: 「🔔 +1 クレジット!」（1,200ms後にクリア）
- フィーバー発動時: 「🔥 FEVER!! 🔥」
- フィーバー終了時: 「FEVER終了」（1,500ms後にクリア）

#### 筐体（cabinet）
- 通常時: 金色ボーダー、金色の光彩
- FEVER中: 赤ボーダー、赤の光彩 + 明滅アニメーション

### 8-3. レスポンシブ対応

| ブレークポイント | リールセル高さ | タイトルフォント |
|----------------|--------------|----------------|
| 通常（≥380px） | 72px | 2rem |
| 小画面（<380px） | 60px | 1.5rem |

- 最大幅: 420px（中央揃え）
- 縦向き（ポートレート）を基本レイアウトとする
